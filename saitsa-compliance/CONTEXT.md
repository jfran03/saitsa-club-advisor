# RAG Procedure

## Step 1 — Identify the topic
Parse the query to extract the key subject (e.g. "grant funding", "event approval", "executive change").

## Step 2 — Scan the File Structure index
Use the File Structure below to identify candidate files before opening anything:
- Match against policy names in /policies-and-procedures
- Match against <Topics> entries in /handbook

## Step 3 — Retrieve from Primary Source first
Read the relevant /policies-and-procedures file(s). This is the authoritative source for rules and requirements.

## Step 4 — Retrieve from Secondary Source if applicable
If a /handbook <Topics> entry overlaps with the retrieved policy, read that file for supplementary context (practical steps, how-tos, examples).

## Step 5 — Synthesize and respond
Combine only what was retrieved. Do NOT add information not present in the files.
If sources conflict, defer to /policies-and-procedures.
Ensure you provide the policy section whenever possible.
You are able to verify, validate, or advise on ideas or tasks based on the writing in the markdown files.
